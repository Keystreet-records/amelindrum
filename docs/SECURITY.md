# Безопасность

## Сразу после клонирования

1. **Смените пароль администратора** в [Supabase Dashboard](https://supabase.com/dashboard) → Authentication → Users.  
   В истории миграций Lovable остались тестовые учётные данные — считайте их скомпрометированными.

2. **Не коммитьте `.env`** — используйте `.env.example` как шаблон.

3. **Ротируйте ключи Supabase / R2**, если они когда-либо попадали в git или общий чат.

## Миграции

- `20260703092524_*` — удаляет **всех** пользователей auth. Не применяйте повторно на проде.
- Сиды с паролем в SQL — только для первичной настройки. В проде создавайте админа через Dashboard и выдайте роль `admin` в `user_roles`.
- Миграции `site-media` (Supabase Storage) **не нужны** — медиа на Cloudflare R2.

## Админ-панель

- `/admin` — `noindex` + роль `admin`.
- Тексты → Supabase `site_content`.
- Фото/видео → Cloudflare R2 (presigned PUT через `/api/upload`), только для залогиненного admin.
- Соцссылки: `http(s)`, `mailto:`, `tel:`; пустой URL скрывает сеть на сайте.
